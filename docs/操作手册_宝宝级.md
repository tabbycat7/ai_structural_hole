# 宝宝级操作手册（零基础也能跑实验）

这份手册假设你**完全不懂代码**。你只要会：打开一个叫「终端 / PowerShell」的黑框框，
然后把手册里的命令**一行一行复制粘贴进去、按回车**，就能跑完整个实验。

> 重要心法：每条命令都是「复制 → 粘贴 → 回车 → 等它跑完出现新的一行光标」。
> 不要一次粘贴好几条。一条跑完了再粘下一条。

---

## 第 0 步：打开那个"黑框框"（终端）

1. 在键盘按 `Win` 键，输入 `powershell`，点开 **Windows PowerShell**。
2. 复制下面这行，粘贴进去，回车。它的作用是「进入我们项目的文件夹」：

```powershell
cd "e:\college\postgraduate\AI_Structural_Holes"
```

3. 如果框框里那行字变成了 `...AI_Structural_Holes>`，说明你进对地方了。

---

## 第 1 步：装好"运行所需的工具"（只需做一次）

复制粘贴回车，等它跑完（可能要几分钟，会刷很多字，正常）：

```powershell
pip install -r requirements.txt
```

跑完后，验证一下是不是装好了（这条会打印因果图，能打印出来就 OK）：

```powershell
python -m ai_structural_holes.cli graph
```

看到一堆 `Q -> S`、`A_S1 = {query_id}` 之类的，就说明环境没问题。✅

---

## 第 2 步：先"空跑"一遍，确认一切正常（不花钱、不用钥匙）

我们做了一个"离线假模型"，可以在**不联网、不花钱**的情况下，把整个流程走一遍，
确认你电脑上一切正常。复制粘贴回车：

```powershell
python -m ai_structural_holes.cli study1 --mock
```

跑完你会看到一张表（每个特征的 EI 和 ATE），最后一行写着
`saved -> ...\outputs\study1`。这表示结果存好了。

> `--mock` 就是"用假模型空跑"的意思。**这一步只是测试，不是真实结果。**

---

## 第 3 步：拿到 OpenRouter 的"钥匙"（API Key）

我们所有真实的大模型（GPT、Claude、Gemini、DeepSeek……）都通过一个叫
**OpenRouter** 的网站统一调用。你只需要在它那里拿一把"钥匙"。

1. 打开网址：https://openrouter.ai ，注册 / 登录（可用 Google 账号）。
2. 充值：点右上角头像 → **Credits** → 充一点钱（比如 5 ~ 10 美元，先小额试水）。
   - 调用大模型是按用量付费的，跑实验会消耗这笔钱（费用见第 6 步）。
3. 拿钥匙：点头像 → **Keys** → **Create Key** → 复制那串以 `sk-or-` 开头的字符串。
   - ⚠️ 这串字符就是你的"钥匙"，**别发给任何人、别截图发群里**。

---

## 第 4 步：把"钥匙"填进项目里（只需做一次）

1. 在项目文件夹里找到一个叫 `.env.example` 的文件。
2. **复制它，重命名为 `.env`**（注意：名字就是一个点加 env，没有别的）。
   - 不会改可以用这条命令自动复制：

```powershell
Copy-Item .env.example .env
```

3. 用记事本打开 `.env`：

```powershell
notepad .env
```

4. 把这一行里的 `sk-or-your-key-here` 替换成你第 3 步复制的真钥匙，然后**保存**：

```
OPENROUTER_API_KEY=sk-or-你复制的那一长串
```

5. 关掉记事本。完成！钥匙只存在你自己电脑上，不会上传。

---

## 第 5 步：用真实模型跑一次小实验（确认钥匙能用）

复制粘贴回车（这条会真的调用 GPT-4o 和 DeepSeek，花一点点钱）：

```powershell
python -m ai_structural_holes.cli study1 --models openai/gpt-4o,deepseek/deepseek-chat
```

- 注意：这次**没有 `--mock`**，所以用的是真模型，结果是真实的。
- 跑完同样会显示一张表 + `saved -> ...\outputs\study1`。
- 如果报错说找不到 key，回第 4 步检查 `.env` 是否保存对了。

---

## 第 6 步：跑"完整实验"（四个 Study 全部）

完整实验 = 依次跑 Study1 ~ Study4。下面给你**直接能用的整套命令**。
你可以一条一条来（推荐），跑完一条再跑下一条。

先设定要测的模型（可按需增减，名字之间用逗号，不要有空格）：

```powershell
$models = "openai/gpt-4o,anthropic/claude-3.5-sonnet,google/gemini-pro-1.5,deepseek/deepseek-chat,qwen/qwen-2.5-72b-instruct"
```

然后依次运行四个实验（`--seeds 3` 表示每个条件重复 3 次取平均，更稳）：

```powershell
python -m ai_structural_holes.cli study1 --models $models --per-domain 2 --seeds 3
```
```powershell
python -m ai_structural_holes.cli study2 --models $models --per-domain 2 --seeds 3 --n-points 16
```
```powershell
python -m ai_structural_holes.cli study3 --models $models --per-domain 2 --seeds 2
```
```powershell
python -m ai_structural_holes.cli study4 --models $models --per-domain 2 --seeds 3
```

每条都会在 `outputs\study1`、`outputs\study2` …… 里存好结果。

### 参数是什么意思（看不懂可忽略，用默认就行）

- `--models`：要对比哪些大模型。
- `--per-domain`：每个领域出几个题目。数字越大，题目越多，越准，但越慢越贵。
- `--seeds`：每个条件重复几次（应对模型回答的随机性）。
- `--n-points`（仅 Study2）：同时改 8 个特征时测多少种组合。
- 想先小成本试：把 `--per-domain` 设 1、`--seeds` 设 1、`--models` 只放 2 个。

### 跑之前先"预估"一下（强烈推荐，不花钱）

在任何命令后面加 `--dry-run`，它**不会真的调用模型、不花一分钱**，只会告诉你这次配置
"要调用多少次、大概多少钱"。例如:

```powershell
python -m ai_structural_holes.cli study1 --models $models --per-domain 2 --seeds 3 --dry-run
```

它会打印类似:`API 调用次数: 7200`、`预计费用(估): ~$14.5 USD`。
看一眼数字觉得能接受，再把 `--dry-run` 去掉真正跑。
（费用是粗略估计，价格按 `--price-in` / `--price-out` 可调;真实单价见 openrouter.ai/models。）

### 大概要花多少钱 / 多少时间？

- 取决于「模型数 × 题目数 × 重复数 × 候选数 × 位置数」，跑的次数会成百上千。
- **强烈建议**：第一次先用**小参数**（`--per-domain 1 --seeds 1` + 2 个模型）跑一遍，
  看一眼花了多少钱（在 OpenRouter 网站 → Credits 里能看到消耗），心里有数后再放大。
- 我们内置了**缓存**：完全相同的请求不会重复花钱，所以重跑很省。

---

## 第 7 步：结果在哪里看？

所有结果都在项目的 `outputs` 文件夹里。用这条命令直接打开它：

```powershell
explorer outputs
```

里面每个 study 一个子文件夹，重点看这几个文件：

- `ei_leverage.csv` / `ei_leverage.png`：**最重要**。每个特征的"结构洞杠杆"排名
  （EI 越高 = 这个特征越能因果地左右模型选不选你）。`.png` 是柱状图，直接看图最直观。
- `ate.csv` / `ate_forest.png`：每个特征让"被选概率"提升/降低了多少。
- `trials.csv`：最原始的逐条记录（每次问模型、它选了谁）。
- Study3 的 `ei_by_model.csv`：不同模型的结论是否一致。
- Study4 的 `deception.csv`：**造假**特征能不能骗到模型（欺骗增益、脆弱性）。

`.csv` 文件可以直接用 Excel / WPS 打开看表格。

## 常见问题（出错了别慌，对照这里）

- **报错含 `OPENROUTER_API_KEY not set`**：钥匙没配好 → 回第 4 步，确认 `.env` 已保存、
  钥匙没填错、没有多余空格。
- **报错含 `401` / `insufficient credits`**：钥匙无效或余额不足 → 去 OpenRouter 检查 Key、充值。
- **某个模型名报错（404 / not found）**：模型名写错了 → 去 https://openrouter.ai/models
  复制正确的模型 slug（就是 `openai/gpt-4o` 这种格式）。
- **跑得很慢**：正常，调用很多次。可以先用小参数；想看进度加 `--progress`。
- **想重跑但不想再花钱**：直接重跑同样命令即可，缓存会自动复用，不会重复扣费。
- **彻底重来（清空缓存/结果）**：删掉 `.cache` 和 `outputs` 两个文件夹即可。

---

## 一句话总结流程

装依赖 → 拿 OpenRouter 钥匙 → 填进 `.env` → 先 `--mock` 空跑确认正常 →
去掉 `--mock` 用小参数试真模型看花费 → 满意后放大参数跑完四个 Study →
去 `outputs` 文件夹看 `ei_leverage.png` 等结果。完事！🎉
