# 效度威胁、控制措施与验收清单

这份文件不是“项目已经全部解决的问题清单”，而是正式报告结果前的审计表。每一项都区分：风险是什么、当前代码做了什么、研究者还需要验证什么。

## 1. 因果结论成立的最低条件

Study 1 要把被选率差异解释为某个特征的因果效应，至少需要满足：

1. 处理文真正实现了目标特征，对照文没有实现该特征。
2. 两篇文章除目标特征外，在事实、立场、相关性、长度和整体质量上足够等价。
3. 两篇文章面对相同竞争者和相同位置分布。
4. 解析失败、模型随机性和重复测量得到正确处理。
5. 分析只比较该特征自己的处理—对照数据。

统计显著不能弥补上述条件的缺失。

## 2. 材料与操纵效度

| 威胁 | 当前实现 | 正式验收要求 | 建议记录 |
| --- | --- | --- | --- |
| 目标特征没有真正实现 | LLM 提示包含目标档位；分析使用 `intended_profile` | 用独立语义评估或人工编码验证实际正文 | 每维命中率、混淆矩阵 |
| 改一个特征连带改变其他特征 | 当前变体不做自动 pair-level 硬门槛 | 对处理—对照逐维检查七个非目标特征 | 漂移维度、漂移率 |
| 事实或主张改变 | 提示要求保持事实内核 | 比较核心事实、答案、立场和推荐对象 | 事实一致率、人工备注 |
| 长度成为真正处理 | 提示要求约 ±10%，但当前不强制 | 计算每对正文长度差；超阈值重做或剔除 | 字数、相对长度差 |
| 模板回退污染 | 空响应会使用 `template_fallback` | 正式样本不混用模板回退；整对重做或剔除 | 各组 generator 比例 |
| 生成器风格泄漏 | 变体由指定生成模型改写 | 固定生成模型；必要时跨生成器复验 | gen_model、base_hash |
| 文章没有回答问题 | 题目只作为生成上下文 | 人工或独立模型评估相关性和完整性 | 相关性评分 |
| 规则关键词替代真实概念 | 基线仍有规则检测；变体不以规则作为最终验收 | S1–S4 使用语义评估；O4 测实际句间距离 | 编码说明、评分者一致性 |

### 硬编码检查应该用在哪里

适合硬编码的项目：

- 是否为空；
- 字数和长度差；
- 是否有列表、小标题、JSON 字段；
- 是否存在模板回退；
- 处理和对照是否共享候选 ID。

不宜只靠关键词判断的项目：

- 证据是否真实和扎实；
- 风险说明是否实质平衡；
- 专业性是否有真实机制，而不是堆术语；
- 结论是否真正明确回答问题；
- 证据是否确实支持并靠近对应主张。

## 3. 竞争环境与任务效度

| 威胁 | 当前实现 | 正式验收要求 |
| --- | --- | --- |
| 处理和对照面对不同竞争者 | Study 1–3 的公共 `assemble()` 为同一问题复用共享干扰文章；Study 4 尚未显式固定三组竞争者 | Study 4 复用同一组干扰文章；在输出或审计表保存并核对候选 ID |
| 位置偏差 | 目标文章遍历所有候选位置 | 报告分位置被选率；确认处理和对照位置完全平衡 |
| 干扰文章过强或过弱 | 支持模板、LLM、真实网页段落 | 正式实验优先使用真实段落，并抽检相关性和质量 |
| 候选数量不符合真实场景 | 默认 `set_size=3` | 在 3、5、8 等规模做稳健性检验，或限制结论范围 |
| 任务只测“强制单选” | 提示要求选择一篇 | 结论限定为单选候选任务，不外推到所有 RAG 行为 |
| 评分理由影响最终选择 | 默认 `minimal` 仅要求 `choice`；`full` 模式才要 ranking/scores/reason | 主分析用 minimal；审计时用 `--output-mode full` |

## 4. 测量与解析效度

| 威胁 | 当前实现 | 正式验收要求 |
| --- | --- | --- |
| 模型输出不是合法 JSON | 解析器支持纯 JSON、代码围栏和完整对象扫描 | 报告总体及处理/对照分组 `parse_ok` |
| 解析失败被误当未选中 | Study 1 主分析剔除 `parse_ok=0` | 其他 Study 也应统一缺失处理，并做敏感性分析 |
| 排名或分数缺失 | `choice` 合法即可产生 Y；其他字段可缺失 | 明确主结局是 choice，次要指标分别报告缺失率 |
| 选择模型实际回退 Mock | 缺密钥时客户端可警告并回退 Mock | 保存运行日志，核对密钥、模型和真实用量 |
| 相同请求被缓存 | 请求哈希命中时复用输出 | 记录缓存策略；不要把缓存重复当成新的独立响应 |
| 解析失败无法回溯 | `Trial` 对象含原始响应，但当前 `trials.csv` 未导出 `raw_response` | 归档原始响应或请求哈希映射，确保能够重新解析 |

解析率不仅要看总体值，还应按以下维度分组：

- 处理/对照；
- 特征；
- 模型；
- 提示风格；
- 候选位置。

如果解析失败与处理条件相关，直接删除也可能产生选择偏差，需要报告敏感性分析。

## 5. 统计结论效度

| 威胁 | 当前实现 | 正式验收要求 |
| --- | --- | --- |
| 把位置和 seed 当独立样本 | Study 1 ATE 按 `query_id` 聚类自助 | 报告独立问题数和聚类区间 |
| EI 混入其他 OFAT 实验 | Study 1、Study 3 均使用 `scope_col="target_dim"` | 对每个特征核对处理/对照样本来自自身配对 |
| ATE 样本数口径误导 | ATE 点估计按 pair 匹配；计数列仍需审计 | 报告有效配对数和问题数，不只引用全局行数 |
| 解析失败进入回归 | Study 1、Study 3 已统一剔除 `parse_ok==0`；Study 2/4 仍未统一 | 正式分析前统一过滤或显式建模缺失 |
| Study 3 EI 口径污染 | 已修：分层 EI（`_ei_by`）与 `cross_model_consistency` 均传 `scope_col="target_dim"` | 复核跨模型/领域 EI 仅用自身配对 |
| 多重比较增加假阳性 | 同时检验多个特征和交互 | 预注册主假设，报告校正后结果或明确探索性 |
| 小样本和完美分离 | 回归失败时可回退 L2 正则 | 正则结果无常规 SE 时不得按普通显著性解释 |
| 分数析因混叠 | Study 2 使用缩减组合 | 检查设计矩阵秩、平衡性和别名结构 |
| EI 排名过度解读 | 输出 EI、归一化 EI、ATE | 同时报告干预概率、ATE、区间，不只报排名 |

### Study 1 的正确分析范围

每个特征必须先筛出自己的数据：

```python
sub = trials[trials["target_dim"] == factor]
```

然后在 `sub` 内比较处理和对照。不能把其他七个实验中恰好具有相同档位的行放入比较组。

### EI 的解释限制

- EI 高表示不同干预档位产生的结果分布差异较大。
- EI 不表示效应为正；方向看 ATE。
- EI_norm 不是方差解释率。
- 排名差异很小时，应报告不确定性，不能硬分第一名和第二名。

## 6. 内部、外部和构念效度

### 6.1 内部效度

主要问题是处理—对照是否只差目标特征。需要：共享基线、共享竞争者、位置平衡、配对检查、按问题聚类。

### 6.2 外部效度

即使内部效度良好，结论也只适用于实际测试的：

- 模型及其当时版本；
- 中文问题和五个领域；
- 候选数量；
- 提示风格；
- 单选引用任务；
- 题库和文章分布。

Study 3 能扩展这些边界，但不能自动代表所有语言、所有 RAG 系统和未来模型。

### 6.3 构念效度

代码中的 S1–S4、O1–O4 是理论概念的操作化。例如，“出现机制一词”不等于真正专业，“出现 N= 和百分比”也不等于证据真实。

正式研究应：

1. 为每个维度制定不依赖生成提示关键词的编码手册；
2. 让至少两名独立评分者盲评一部分材料；
3. 报告 Cohen's kappa、Krippendorff's alpha 或其他一致性指标；
4. 根据评分结果修订材料，而不是只修订检测器。

## 7. Study 4 的特殊真实性要求

Study 4 的关键不是“模型是否写出了 fake 标签”，而是研究者能否证明材料真的属于三种状态。

### genuine 组

- 来源可以访问或有可靠存档；
- 数字和来源之间对应；
- 证据确实支持文中主张；
- 专业概念使用正确。

### fake 组

- 伪造内容经过记录，不会被误认为真实来源；
- 与 genuine 组在长度、语气、格式和表面可信度上尽量匹配；
- 主要差异是可核验真实性，而不是明显荒谬程度。

### none 组

- 不含目标证据或专业性特征；
- 仍需保持基本可读和问题相关，不能故意写成低质量文章。

如果 fake 组明显更短、更夸张或更不相关，测到的是写作质量差异，不是模型的事实核验能力。

## 8. 正式报告前的验收门

以下项目没有完成时，建议将结果标为 pilot 或 exploratory，而不是 confirmatory：

- [ ] 题库和真实段落完成抽检并冻结。
- [ ] 基线文章不存在未处理的模板回退。
- [ ] 变体文章完成独立语义质检。
- [ ] 处理—对照的非目标维度漂移率在预设阈值内。
- [ ] 长度差和生成路线在处理/对照之间平衡。
- [ ] 处理和对照共享完全相同的竞争者。
- [ ] Study 4 的 none/genuine/fake 共享完全相同的竞争者。
- [ ] 候选位置计数平衡。
- [ ] 解析率及其组间差异已报告。
- [ ] 解析失败处理规则已在所有 Study 中统一。
- [ ] ATE 按问题聚类并报告有效问题数。
- [ ] OFAT 的 EI 只使用对应 `target_dim`。
- [ ] Study 2 设计矩阵和交互可识别性已检查。
- [ ] Study 3 的分层 EI 口径已复核。
- [ ] Study 4 的 genuine/fake 经过独立真实性验证。
- [ ] 代码版本、命令、材料和输出均已归档。

## 9. 建议的最低报告内容

论文或报告至少应包含：

- 每个领域的问题数和有效配对数；
- 每个模型、提示风格、位置的 trial 数；
- 变体生成模型、模板回退率和文章长度分布；
- 操纵成功率和非目标维度漂移率；
- 总体及分组解析率；
- 处理/对照干预概率、ATE、聚类置信区间；
- EI 及其计算范围；
- 跨模型、跨领域方向一致性；
- 所有剔除规则和敏感性分析；
- 代码、材料和模型版本信息。

最稳妥的原则是：**先证明实验确实操纵了想研究的东西，再讨论模型为什么做出不同选择。**
