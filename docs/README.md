## 推荐阅读顺序

1. 先读“四个实验”，建立整体画面。
2. 再读“实验说明”，理解为什么这样设计和怎样解释 ATE、EI。
3. 真正运行实验时，只按“操作手册”执行。
4. 在正式跑数和写论文结论前，对照“效度威胁清单”逐项验收。

## 三种内容要分清

本文档会明确区分以下三类内容：

- **研究设计**：理论上希望实验做到什么。
- **当前实现**：现有代码实际上会做什么。
- **正式实验要求**：如果要把结果写成论文结论，还必须额外检查什么。

这一区分很重要。例如，设计上要求处理文和对照文“只相差一个特征”；当前代码会给生成模型这样的改写指令，但不会自动证明最终正文真的只改变了该特征。因此，正式实验仍需要独立语义质检和人工抽检。

## 当前最重要的使用原则

- `--mock` 和模板文章只用于测试流程，不能产生真实研究结论。
- 正式实验应冻结题库、基线文章和变体文章，并保留完整版本记录。
- LLM 变体当前没有硬编码规则作为最终验收；`intended_profile` 是设计标签，不等于已经验证的实际文本特征。
- 变体为空时会回退模板。正式实验不应把模板回退文和正常 LLM 变体混在同一分析中。
- Study 1 的 ATE、EI 只能在对应特征自己的配对数据内计算；解析失败应作为缺失记录审计。
- EI 表示干预特征对选择结果的区分能力，不表示“解释了百分之多少”，也不提供效应方向；方向要看 ATE。

## 项目主要目录

```text
data/query_pool/        冻结的真实问题和网页段落
data/base_articles/     每个问题的冻结基线文章
data/variant_articles/  按设计档位生成并冻结的变体文章
outputs/study*/         每个实验的逐次记录、统计表和图
.cache/llm/             模型请求缓存
```

如果代码行为与文档描述不一致，以代码和实际输出为准，并及时同步修订文档。
